# Koray-Style SEO Playbook for a Local Directory

## 1) Build a topical map around local intent
Define clear clusters (service + city, comparisons, problem-intent, process-intent) so pages are organized by user intent, not listing volume.

## 2) Decide what should be indexed
Index only pages with unique, useful value. Noindex/canonicalize thin, duplicate, or low-demand combinations (especially faceted/filter URLs).

## 3) Increase listing depth with entity-attribute completeness
For each listing, enrich structured details: services, areas served, hours, pricing range, proof signals, differentiators, and FAQs.

## 4) Strengthen hierarchy and internal links
Use a strong structure: city hubs -> service pages -> supporting pages -> listings. Add relevant cross-links to reinforce context and discoverability.

## 5) Add information gain on key pages
Include unique editorial insights (local pricing patterns, neighborhood nuances, selection criteria, comparisons) so pages offer more than commodity directory data.

## 6) Control crawl paths and URL sprawl
Prevent index bloat from filter combinations and near-duplicate URLs. Keep sitemaps focused on canonical, index-worthy pages.

## 7) Expand in quality-first waves
Start with a focused city/vertical set, validate performance, then scale to adjacent areas only after quality thresholds are met.

## 8) Track the right KPIs by page type
Monitor indexation rate, impressions/clicks, crawl/index coverage statuses, ranking trends, and conversion actions (calls, click-outs, leads).

## 9) Avoid common implementation mistakes
Don’t publish massive thin inventories too early, don’t confuse semantics with keyword variation, and don’t treat crawl budget as the only issue when page quality is low.
